平衡树之Treap学习笔记

发表于 2020-07-02 | 分类于 学习笔记

Treap 是一种平衡树。可以做到插入一个数值、删除一个数值，查询一个数值前驱后继、查询一个数值排名、查询指定排名的数值等等。

前置知识：BST（二叉搜索树）。

Treap

BST 很早就学过了，其是一个完全二叉树。

开头提到的功能，它也能做到。

但是，如果插入的数不是随机的话，可能退化为链，这样每次操作的时间复杂度会退化为 $O(n)$。

因此，“平衡树”的概念就出现了。顾名思义，就是“平衡”的 BST。

先定义如下变量。

每种平衡树变得平衡的操作不同。

这次学的 Treap，Tree+heap，即为结合堆来使其变得平衡。

如何结合呢？

可以将每个节点附加一个“额外权值”，这个值是随机的。

每个节点也有本身的权值，称其为”关键码“。

根据中序遍历为同一序列的树，有很多不同的形态，那么如果使得它平衡的话，自然是完全二叉树比较好。

因此，保持权值满足 BST 性质，保持额外权值满足堆性质，那么就会变得平衡了！

那么如何使额外权值满足堆性质呢？

引入两个操作：左旋（lturn）与右旋（rturn）。

借用 史上最强图解Treap总结， 不是浅谈！ 的两个图。

可以发现，左旋一个点 x 后，rs(x) 替换到了原来 x 的位置，而 x 则变为了左子节点。

右旋则相反。

因此，可以理解为，左/右旋一个节点，用其右/左子节点替换其位置，而其则变为左/右子节点。

那么什么时候需要进行左右旋呢？

当然是操作的时候了。。

考虑插入一个数值 $val$ 的操作，有以下几种情况。

  1. 当前节点编号为 $0$，则说明树中没有该关键码的节点，新建一个关键码为 $val$ 的节点。
  2. 当前节点编号不为 $0$。
    1. 当前节点关键码等于 $val$。则将该结点副本数加一，并 update 该点。
    2. 当前节点关键码大于 $val$。则递归到左子节点。之后如果左子节点的额外权值大于当前节点的额外权值，那么就对当前节点进行右旋。
    3. 当前节点关键码小于 $val$。则递归到右子节点。之后如果右子节点的额外权值大于当前节点的额外权值，那么就对当前节点进行左旋。

最后再 update 一下。

考虑删除一个数值 $val$ 的时候，有以下几种情况。

  1. 当前节点编号为 $0$，则直接回溯。

  2. 当前节点编号不为 $0$。

    1. 当前节点的关键码等于 $val$。

      1. 当前节点的副本数大于 $1$。则直接将副本数减一然后 update 当前节点。

      2. 当前节点的副本数等于 $1$。

        1. 存在子节点。如果不存在右节点或者左节点的额外权值大于右节点的额外权值，那么对该节点进行右旋，然后递归到右子节点；反之则对该节点进行左旋，然后递归到左子节点。
        2. 不存在子节点。直接将该节点的编号赋为 $0$。

        最后 update。

    2. 当前节点的关键码大于 $val$。则递归到左子节点。

    3. 当前节点的关键码小于 $val$。则递归到右子节点。

最后 update。

考虑获取一个数 $val$ 的排名，有以下几种情况。

  1. 当前节点编号为 $0$，则返回 $0$。
  2. 当前节点编号不为 $0$。
    1. 当前节点的关键码等于 $val$，则直接返回 s(ls(p))+1。
    2. 当前节点的关键码大于 $val$，则返回递归到左子节点的返回值。
    3. 当前节点的关键码小于 $val$，则返回递归到右子节点的返回值加上 s(ls(p))+c(p)。

考虑获取排名为 $rank$ 的数，有以下几种情况。

  1. 当前节点的左子节点的 size 值大于等于 $rank$，则返回递归到左子节点的返回值。
  2. 当前节点的左子节点的 size 值加上当前节点的副本数大于等于 $rank$，则直接返回当前节点的关键码。
  3. 当前节点的左子节点的 size 值加上当前节点的副本数小于 $rank$，则将 $rank$ 减去当前节点的左子节点的 size 值与当前节点的副本数，然后返回递归到右子节点的返回值。

考虑查询一个数 $val$ 的前驱，有以下几种情况。

  1. 当前节点编号为 $0$，则返回 -INF。
  2. 当前节点的关键码大于等于 $val$，则返回递归到左子节点的返回值。
  3. 当前节点的关键码小于 $val$，则返回递归到右子节点的返回值与当前节点的关键码的较大值。

考虑查询一个数 $val$ 的后继，有以下几种情况。

  1. 当前节点编号为 $0$，则返回 INF。
  2. 当前节点的关键码小于等于 $val$，则返回递归到右子节点的返回值。
  3. 当前节点的关键码大于 $val$，则返回递归到左子节点的返回值与当前节点的关键码的较小值。

至此，大部分操作就讲完了。

但是仍有注意点。

在查询一个数 $val$ 排名的时候，要对答案减一，因为初始建树的时候新建了一个关键码为 -INF 的节点。

在查询排名为 $rank$ 的数的时候，要对 $rank+1$，原因同上。
